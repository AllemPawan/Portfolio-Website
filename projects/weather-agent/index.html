Weather Agent

An LLM-powered weather assistant that understands natural language queries, supports multiple cities, and provides current conditions and forecasts via tool calling.

Python LangChain FastAPI Ollama Docker
View on GitHub

Overview

The Weather Agent is an LLM-powered conversational assistant that provides weather information through natural language interactions. It uses function/tool calling to query a weather API, interprets the results, and presents them in a conversational format.

Ask about current conditions, forecasts, or compare weather across multiple cities — the agent handles the underlying API calls and data formatting automatically. Built with conversation memory for natural multi-turn interactions.

Features

🌤️

Weather Tool

Integrated weather API tool that the LLM calls automatically when needed.

💬

Natural Language Queries

Ask in plain English — "What's the weather in Tokyo?" or "Do I need an umbrella tomorrow?"

🏙️

Multi-City Support

Query and compare weather across multiple cities in a single conversation.

🌡️

Current Weather

Real-time conditions including temperature, humidity, wind speed, and description.

📅

Forecast

Multi-day forecast with highs, lows, precipitation chances, and conditions.

🔧

Tool Calling

LLM decides when to invoke weather tools based on user intent.

⚡

Streaming Responses

Token-by-token streaming for real-time assistant responses.

🧠

Conversation Memory

Remembers context across turns — refer to previous cities or queries naturally.

🐳

Docker Support

Containerized deployment with all dependencies and API keys configured.

Getting Started

# Clone & run
git clone https://github.com/AllemPawan/weather-agent.git
cd weather-agent

# Add your Weather API key to .env
echo "WEATHER_API_KEY=your_key_here" > .env

docker compose up -d

# Open http://localhost:8000
# Ask: "What's the weather in Tokyo?" or "Do I need a jacket in London?"
